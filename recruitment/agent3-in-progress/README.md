# 🔍 招聘策略师

> AI招聘决策系统 · Agent 3 / 3

不是给你更多面试题，而是帮你找到**最精准的验证角度**。基于岗位要求和候选人背景，设计差异化的面试、测评和录用判断方案，避免把"平台光环"错当成"可迁移能力"。

---

## 它解决什么问题？

人才画像师帮你定义"到底要什么人"，雇主镜像师帮你检查"对的人愿不愿意来"。  
招聘策略师解决的是第三个关键问题：

**候选人已经坐到面试桌对面了，怎么判断他是不是真的能做、能不能迁移、值不值得拿下？**

很多招聘失误，不是因为题不够多，而是因为验证角度错了：

- 把大厂背景当成真实能力
- 把过往成绩当成可复制成绩
- 只测"会不会说"，没测"能不能做"
- 只看不用AI的裸能力，没看与AI协作后的产出天花板
- 没有识别候选人从A环境迁移到B环境时的折损风险

招聘策略师的价值，就是把面试从"凭感觉聊天"，变成一套**可验证、可交叉校验、可落地决策**的评估方案。

## 快速上手

**第一步：打开 Bot**  
前往扣子平台，搜索「招聘策略师」或「HR小酒馆」，找到对应 Bot。

**第二步：准备材料**

| 材料 | 重要程度 |
|------|---------|
| 人才画像师（Agent 1）报告 | ⭐⭐⭐ 最佳输入 |
| 岗位 JD 全文 / 岗位描述 | ⭐⭐⭐ 必须有其一 |
| 公司行业、规模、招聘背景 | ⭐⭐ 建议提供 |
| 候选人简历 / 背景信息 | ⭐⭐ 有候选人时强烈建议 |

> 最佳用法：先用人才画像师定义岗位标准，再把结果喂给招聘策略师做面试设计。这样输出最稳，也最不容易跑偏。

**第三步：按场景进入对应流程**

| 场景 | Agent怎么工作 |
|------|---------------|
| 只有岗位信息 | 先输出考察菜单：核心验证点 + 推荐测评组合 |
| 已经准备设计完整面试 | 继续拆成轮次方案、题目风格和评分标准 |
| 已经有具体候选人 | 追加候选人定制：迁移风险、红旗检测、决策建议 |
| 只是临时问一个问题 | 走轻量查询模式，快速给出几种考法 |

## 三阶段流程

### 阶段一：考察菜单

轻输入，先出结果。  
只要你提供岗位信息，Agent就先给你：

- 5-6个最关键的核心验证点
- 每个验证点为什么重要
- 每个验证点最推荐的验证方式
- 每个验证点1道代表性预览题
- 推荐的考核形态组合
- AI允许环节（测天花板）+ AI不允许环节（测地板）

这一阶段的目的不是一次性给完所有题，而是先帮你看清楚：**这个岗位到底该考什么、先考什么、值不值得加别的测评方式。**

### 阶段二：框架拆分

当你确认了菜单，Agent再把它拆成真正可执行的面试框架：

- 几轮面试最合理
- 每轮目标是什么
- 每轮时间怎么分配
- 哪些验证点放在哪一轮
- 每个验证点给出不同风格的题目选项
- 每道题都带考察目的、好回答参考、差回答信号和追问方向
- 同一个能力如何跨轮次交叉验证

这一步的重点不是"题库更大"，而是让面试官知道：**每一道题到底在测什么，问完以后怎么判断。**

### 阶段三：候选人定制

有候选人时，Agent会在通用框架上做定制化调整：

- 判断候选人属于哪类迁移场景
- 评估环境迁移风险
- 用STAR第五维做"环境剥离追问"
- 扫描最多3个需要重点验证的红旗信号
- 调整哪些环节对这个候选人最关键
- 输出通过标准、待定条件、否决项和试用期关注点
- 补一版简化的Offer提示

这也是招聘策略师和普通面试工具最不一样的地方：  
**它不是只帮你设计岗位面试，而是帮你判断“这个人离开原来的土壤后，还剩多少真能力”。**

### 模式C：轻量查询

如果你只是想问一个具体问题，比如：

- 怎么考察数据分析能力？
- 找茬测试怎么设计？
- 项目管理能力除了面试还能怎么测？

Agent会直接给你2-3种考法对比，再按你的选择展开，不需要走完整流程。

---

## 这个 Agent 的核心价值

1. **不是出更难的题，而是找最准的验证角度。** 真正决定招聘质量的，不是题目数量，而是你有没有抓到那个岗位最关键、最容易伪装、最值得交叉验证的点。
2. **把“能力”拆成“可迁移”和“不可迁移”。** 同样的成绩，在大平台、强品牌、成熟团队和小公司里，含金量完全不同。招聘策略师会主动追问环境依赖性。
3. **同时测天花板和地板。** 允许AI的环节测候选人与AI协作后的产出上限，不允许AI的环节测他的裸能力底盘。少了任何一边，判断都会失真。
4. **让面试官从“会聊天”变成“会验证”。** 每道题都配判断标准、追问方向和差回答信号，减少面试失控和主观漂移。
5. **把结果接到决策。** 最终输出不是“整体还不错”，而是通过标准、待定条件、否决项、试用期关注点和简版Offer提示。

---

## 会产出哪些内容？

1. 核心验证点清单（按优先级排序）
2. 推荐考核形态组合
3. AI天花板/地板双测建议
4. 按轮次拆好的面试计划
5. 多风格面试题库（每题带考察目的和判断参考）
6. 评分标准与跨轮次交叉验证设计
7. 候选人迁移风险评估
8. 红旗信号检测与专项验证题
9. 录用决策建议
10. 简化版Offer提示与入职关注点

## 当前版本局限

**仍处于草稿迭代期**  
当前 `prompt.md` 已经搭好了方法论骨架，但具体知识库、案例和对话细节还会继续补充。

**输入质量决定输出上限**  
如果只有一个模糊岗位名称，没有JD、没有核心任务、没有候选人背景，Agent仍然能给结构，但精准度会明显下降。

**不替代用人判断**  
Agent能帮你把验证逻辑搭出来，但最终是否录用、哪些风险可接受、Offer怎么谈，仍然需要HR和业务负责人共同判断。

**候选人定制依赖材料完整度**  
环境迁移评估、红旗检测、决策建议这些高价值模块，前提是你给到足够具体的候选人背景信息。
